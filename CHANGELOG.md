## [2.0.0] - 2026-02-07

### ðŸš€ Major Architecture Overhaul: Typst-Based Rendering

**Rescume v2.0.0** replaces the DOCX-based formatting pipeline with a Typst-based PDF rendering system, eliminating layout responsibility from the LLM and ensuring deterministic, single-page output.

### Breaking Changes

- **Output Format**: Changed from DOCX to PDF (generated by Typst)
- **Content Format**: LLM now generates JSON instead of formatted DOCX
- **Removed Skills**: `word-counter` (no longer needed)
- **Removed Agents**: `compression-strategist` (replaced by auto-fit templates)
- **Updated Agent**: `content-generator` now outputs pure JSON without formatting

### Added

#### New Typst Rendering System
- **`typst-renderer` Skill**: Complete PDF compilation pipeline
  - `compile.py`: Orchestrates JSON â†’ Typst â†’ PDF conversion
  - `json_to_typst.py`: Converts JSON to Typst data format
  - `validate_pdf.py`: Validates PDF page count and content
  - `list_templates.py`: Lists available Typst templates

#### Template System
- **`templates/` Directory**: Modular Typst resume templates
- **`simple-technical-resume`**: First template based on Typst Universe package
- **Auto-Fit Logic**: Templates automatically adjust font size to fit content on one page
- **Template Metadata**: JSON metadata for each template (features, fonts, recommendations)

### Changed

#### Core Workflow
- **Phase 2 Simplified**: Removed iterative compression loops
  - LLM generates content â†’ Typst compiles â†’ Done (typically <200ms total)
  - If content overflows: trim 2-3 bullets â†’ recompile (no complex compression strategy)

#### Agent Updates
- **`content-generator`**:
  - Now outputs structured JSON instead of DOCX
  - Removed all word-counting logic
  - Removed formatting instructions
  - Focus purely on content quality
  - Added JSON schema definition
- **`hr-critic`**:
  - Removed "triage" mode (no longer needed)
  - Only evaluates content quality, not formatting
  - Removed word count/page fitting concerns

### Removed

- **`word-counter` Skill**: Auto-fit templates handle page fitting
- **`compression-strategist` Agent**: Auto-fit eliminates need for iterative compression
- **DOCX Output Pipeline**: Replaced by Typst PDF generation
- **Configuration Options**:
  - `rescume.targetWordCount` (no longer applicable)
  - `rescume.maxCompressionIterations` (no longer applicable)

### Improved

- **Deterministic Rendering**: Same content always produces identical PDF
- **Performance**: ~50-100ms Typst compilation (vs slow DOCX â†’ PDF conversion)
- **LLM Focus**: AI focuses on content, not layout engineering
- **Guaranteed Page Fit**: Auto-fit templates ensure single-page output
- **No Layout Bugs**: Eliminates "style drift" and formatting inconsistencies

### Dependencies

#### Added
- **Typst CLI** (v0.11+): PDF rendering engine
  - Install: `brew install typst` (macOS) or visit https://typst.app
- **pdfplumber**: PDF validation (`pip install pdfplumber`)

#### Changed
- **python-docx**: Now only for Phase 1 DOCX parsing (not output generation)

### Migration Notes

**For existing users:**
- Resume database format unchanged (backward compatible)
- Phase 1 (parsing) workflow unchanged
- Phase 2 (tailoring) now outputs PDF instead of DOCX
- Install Typst CLI: `brew install typst`
- Re-run tailoring to generate PDF output

**For contributors:**
- See `RESCUME_V2_UPDATE_PLAN.md` for detailed architecture changes
- Template authoring guide: `templates/README.md`
- Typst renderer documentation: `skills/typst-renderer/SKILL.md`

## [1.0.0] - 2026-02-05

### ðŸŽ‰ Initial Release

**Rescume v1.0.0** - AI-Powered Resume Tailoring System

### Added

#### Core Features
- **7 Specialized AI Agents**:
  - `resume-parser`: Parse DOCX resumes into structured JSON
  - `ats-analyzer`: Analyze job descriptions for ATS keywords
  - `coverage-mapper`: Map experiences to job requirements
  - `content-generator`: Generate tailored resume content
  - `hr-critic`: Evaluate resume quality (3 modes: comprehensive, triage, final validation)
  - `compression-strategist`: Optimize word count to fit one page
  - `interview-conductor`: Generate intelligent follow-up questions

- **4 Custom Skills**:
  - `rescume`: Main coordinator orchestrating the workflow
  - `json-database`: Resume data management (5 Python scripts)
  - `word-counter`: Section-by-section word counting
  - `coverage-tracker`: Skill coverage verification

- **Database System**:
  - Persistent JSON storage for resume data
  - Automatic ID generation
  - Database validation and integrity checks
  - Backup system for data safety

- **Resume Optimization**:
  - Section-based word count tracking (Header, Education, Experience, Skills)
  - Target: 475 words for one-page format
  - Only edits Experience and Skills sections
  - Preserves user's original template formatting

- **Quality Assurance**:
  - HR Critic evaluation (1-10 scoring)
  - Hire probability estimation
  - ATS keyword optimization
  - 100% required skill coverage guarantee

- **User Experience**:
  - Natural language triggering ("tailor my resume for...")
  - Custom slash commands (`/rescume start`, `/rescume parse`, `/rescume analyze`)
  - Interactive follow-up questions
  - User validation loop before finalizing

#### Custom Commands
- `/rescume start`: Start tailoring workflow
- `/rescume parse`: Parse resume into database
- `/rescume analyze`: Analyze job description

#### Lifecycle Hooks
- `onInstall`: Initialize database and install dependencies
- `onUpdate`: Migrate database and backup data
- `onUninstall`: Clean up with user confirmation

#### Configuration Options
- `rescume.targetWordCount`: Target word count (default: 475)
- `rescume.qualityThreshold`: Minimum quality score (default: 7.0)
- `rescume.hireProbabilityThreshold`: Minimum hire probability (default: 0.70)
- `rescume.maxCompressionIterations`: Max compression iterations (default: 10)
- `rescume.databasePath`: Database location (default: data/comprehensive_db)
- `rescume.autoBackup`: Auto-backup database (default: true)

### Technical Details

- **Python Version**: 3.8+
- **Dependencies**: python-docx >= 0.8.11
- **File Format**: DOCX input/output (PDF not supported)
- **Storage**: Local JSON files (~50MB for plugin + variable for user data)

### Workflow

**Phase 1: Initial Setup (One-Time)**
1. User uploads DOCX resume(s)
2. Resume Parser extracts structured data
3. Interview Conductor asks enrichment questions
4. Comprehensive database built

**Phase 2: Tailoring (Per Job)**
1. User provides job description
2. ATS Analyzer extracts requirements
3. Coverage Mapper checks skill coverage
4. Interview Conductor fills gaps (if needed)
5. Content Generator creates draft (~550-600 words)
6. HR Critic evaluates and provides feedback
7. Compression Strategist optimizes to ~475 words
8. HR Critic final validation
9. Deliver DOCX + PDF

### Known Limitations

- DOCX files only (PDF conversion not supported)
- Requires manual user validation of page count
- One-page format only (multi-page support planned)
- English language only (multilingual support planned)
